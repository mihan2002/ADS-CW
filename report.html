1. Executive Summary

This document presents the overarching architectural structure of the Backend API built using TypeScript, Express.js, and Drizzle ORM.

The system is designed with a layered Service-Oriented MVC Architecture, facilitating maintainability and scalability. The database is relational (MySQL), optimized for structured data and complex relationships involving User identities, Alumni profiles, Sponsorship offers, and Feature Day bidding.

Core architectural principles:

  • Separation of Concerns: Routing, controllers, and business logic are fully decoupled.
  • Robust Data Validation: Utilizing Zod schemas for Data Transfer Objects (DTOs).
  • Security Best Practices: Encrypted tokens, hashed passwords, and strict cascading deletes at the database layer.

4. MVC (Service-Controller) Architecture

The application uses an extended MVC pattern where business logic is moved out of the controllers into a dedicated Service Layer. DTOs validate data between layers.

classDiagram
    class Route {
        <<Router>>
        +UserRoutes
        +AuthRoutes
        +AlumniRoutes
    }
    class Middleware {
        +AuthMiddleware()
        +ValidationMiddleware(DTO)
        +ErrorHandler()
    }
    class Controller {
        <<ControllerLayer>>
        +Login()
        +GetProfile()
        +PlaceBid()
    }
    class Service {
        <<BusinessLogic>>
        +verifyCredentials()
        +fetchUserData()
        +processBidTransaction()
    }
    class Schema {
        <<DrizzleEntities>>
        +usersTable
        +alumniProfilesTable
        +bidsTable
    }
    
    Route --> Middleware : Validates and Secures
    Middleware --> Controller : Passes Request
    Controller --> Service : Calls Method
    Service --> Schema : Queries or Mutates Data

5. Database Schema: Users & Authentication

The foundation of the schema consists of the User table and its associated temporary tokens for Email Verification and Password Resets.

9. Key Endpoints: Authentication & Users

Method Endpoint Description
POST/api/auth/loginAuthenticates user and returns JWT tokens.
POST/api/auth/refreshRefreshes the access token using a refresh token.
POST/api/auth/verify-emailVerifies email via 6-digit OTP token.
POST/api/auth/reset-passwordResets password after valid request.
GET/api/usersFetches all users.
POST/api/usersRegisters a new root user identity.
